2.下载Linux镜像

镜像下载地址

centos-8-stream-isos-x8664安装包下载开源镜像站-阿里云 (aliyun.com)

Ubuntu 20.04 server版镜像

这里我选择的是ubuntu，因为它更适合搭配docker使用，而我主要就是用docker来搭建各种服务，所以选择了它。

3.安装虚拟机服务器（Ubuntu版）

1.选择自定义安装

2.默认下一步

3.选择稍后安装操作系统

4.选择Linux，版本选择Ubuntu 64位

5.默认下一步

6.这里有说法，处理器数量*处理器内核总数 <= 自己电脑CPU的逻辑处理器数量（任务管理器可以看），我这里选择自己CPU内核总数的一半

7.这里内存我选择自己电脑内存的一半

8.后面一路默认，直到分配磁盘容量，建议30个G

9.后面也是一路默认，到自定义硬件时，点开新CD/DVD这一栏，在右侧加载下载好的iso镜像，最后点击关闭完成

4.启动虚拟机服务器（Ubuntu版）

1.选择语言，没有中文，选择English

2.提示更新版本，默认就是不更新，继续

3.键盘布局，默认就行，继续

4.配置动态IP地址，服务器实际使用的是静态IP，这里为了能让虚拟机能够直接访问外网下载软件包，先默认使用DHCP连接，后面再改成静态ip

5.服务器代理，不需要设置，继续

6.镜像源地址，需要设置为阿里云镜像(https://mirrors.aliyun.com/ubuntu/)

7.配置硬盘，默认就行

8.配置文件系统，还是默认就行，回车Done后会跳出一个提示框，选择继续就行

9.配置账户，第一个是使用者姓名，第二个是服务器名字，第三个账户名，第四个是账户密码

10.安装SSH连接工具，这个必须要安装，按下空格显示x则表明选中

11.安装其他工具，暂时不安装，直接下一步会进行系统安装，需要等待一会儿

12.内核安装完成后，等待安装一些更新，若因为网络问题太慢可以选择取消更新，直接选择Reboot重启，至此服务器已经安装完成！

6.搭建服务端所需依赖

Docker

简介

Docker依赖于已存在并运行的Linux内核环境，Docker实质上是在已经运行的Linux下制造了一个隔离的文件环境，因此它执行的效率几乎等同于所部署的Linux主机。因此，Docker必须部署在Linux内核的系统上，如果其他系统想要部署Docker就必须安装一个虚拟Linux系统。

Docker由三部分组成，仓库、镜像、容器，仓库存放多个镜像，一个镜像可以创建多个容器。

架构

C/S模式架构，Docker的运行流程如下：

1.用户使用 Docker Client 与 Docker Daemon 建立通信，并发送请求给后者

2.Docker Daemon作为Docker架构中的主体部分，首先提供 Docker Server 的功能使其可以接收 Docker Client 的请求

3.Docker Engine执行Docker内部的一系列工作，每一项工作都是以一个Job形式存在

4.Job的运行过程中，当需要容器镜像时，则从 Docker Registry 中下载镜像，并通过镜像管理驱动 Graph Driver 将下载镜像以Graph的形式存储

5.当需要为Docker创建网络环境时，通过网络管理驱动 Network Driver 创建并配置Docker容器网络环境

6.当需要限制Docker容器运行资源或执行用户指令等操作时，则通过Exec Driver来完成

7.Libcontainer时一项独立的容器管理包，Network Driver以及Exec Driver都是通过Libcontainer来实现具体对容器进行的操作
